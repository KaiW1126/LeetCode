# 🟣 動的計画法（Dynamic Programming）

## どんなアルゴリズム？

大きな問題を **小さな部分問題に分割** し、各部分問題の答えを **テーブル（配列）に保存** しながら積み上げていく手法。
一度計算した結果を再利用するので、同じ計算を何度もやる無駄がなくなる。

**イメージ**: 「メモしながら解くパズル」 — 一度解いたピースの答えをメモしておき、次に同じピースが出てきたらメモを見返す。

---

## こう聞かれたらDP！

| キーワード | 例 |
|---|---|
| **「何通りありますか？」** | Decode Ways, Unique Paths, Climbing Stairs |
| **「最大/最小の〇〇は？」**（累積系） | Maximum Subarray, Coin Change |
| **「可能かどうか？」**（True/False） | Word Break, Jump Game |

---

## 見分けるコツ

```
✅ DPの3つのサイン:
  1. 「i番目の答え」が「i-1番目やi-2番目の答え」から計算できる
  2. 問題に「重なる部分問題」がある（同じ計算を何度もする）
  3. 最適解を求める（最大・最小・数え上げ）

❌ DPではないサイン:
  ・全パターンを列挙する必要がある → バックトラッキング
  ・データ構造の操作がメイン → スタック/キューなど
```

---

## DPの考え方テンプレート

```
1. dp[i] の定義を決める（「〇〇のときの答え」）
2. 遷移式を見つける（dp[i] = dp[i-1] + dp[i-2] など）
3. 初期値を設定する（dp[0] = ?, dp[1] = ?）
4. ループで埋めて、最後のdpを返す
```

---

## Python での書き方

### 基本テンプレート（1次元DP）
```python
def solve(n):
    # 1. DP配列を初期化
    dp = [0] * (n + 1)

    # 2. 初期値を設定
    dp[0] = 1  # 基底ケース

    # 3. 遷移式でループ
    for i in range(1, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]  # 例: フィボナッチ

    # 4. 答えを返す
    return dp[n]
```

### 空間最適化（変数2つだけ）
```python
def solve(n):
    if n <= 2:
        return n
    prev2, prev1 = 1, 2
    for _ in range(3, n + 1):
        current = prev1 + prev2
        prev2 = prev1
        prev1 = current
    return prev1
```

### True/False DP テンプレート
```python
def solve(s, wordDict):
    dp = [False] * (len(s) + 1)
    dp[0] = True  # 空文字列は常にTrue

    for i in range(1, len(s) + 1):
        for j in range(i):
            if dp[j] and s[j:i] in wordDict:
                dp[i] = True
                break
    return dp[len(s)]
```

### Python 構文ポイント
```python
# DP配列の初期化
dp = [0] * (n + 1)           # n+1個の0で初期化
dp = [False] * (len(s) + 1)  # True/False のDP

# +1 する理由: dp[0]〜dp[n] まで使うので n+1 個必要
# dp[0] は「空」= 基底ケース
```

## 解いた問題

### Decode Ways（数え上げDP）
> **「何通りデコードできますか？」→ 数え上げ → DP！**

`dp[i]` = i文字目までのデコード数。1文字デコードと2文字デコードの両方を試す。

📂 [DecodeWays.py](../Blind75/Python/Medium/DecodeWays.py)

---

### Word Break（True/False DP）
> **「辞書の単語だけで分割できますか？」→ 可能か判定 → DP！**

`dp[i]` = 先頭i文字を辞書の単語で分割できるか。`dp[j]`がTrueかつ`s[j:i]`が辞書にあれば`dp[i] = True`。

📂 [WordBreak.py](../Blind75/Python/Medium/WordBreak.py)

---

### Maximum Subarray（カダネのアルゴリズム）
> **「最大の連続部分配列の和は？」→ 最大値 → DP！**

現在の合計が負になったらリセット。`dp[i] = max(nums[i], dp[i-1] + nums[i])`と同義。

📂 [MaxSubarray.py](../Blind75/Python/Medium/MaxSubarray.py)

---

### Unique Paths（経路数え上げ）
> **「左上から右下への経路は何通り？」→ 数え上げ → DP！**

組み合わせの数学的解法 `C(m+n-2, m-1)` でも解ける。DPなら `dp[i][j] = dp[i-1][j] + dp[i][j-1]`。

📂 [UniquePaths.py](../Blind75/Python/Medium/UniquePaths.py)

---

### Min Cost Climbing Stairs（最小コストDP）
> **「最小コストで頂上に到達するには？」→ 最小値 → DP！**

`dp[i] = cost[i] + min(dp[i-1], dp[i-2])`。最後は `min(dp[-1], dp[-2])` を返す。

📂 [MInCostStairs.py](../NeetCode150/easy/MInCostStairs.py)

## よく間違えるパターン

| この問題 | 間違えやすい | 正解 | なぜ？ |
|---|---|---|---|
| Jump Game | DP | 🟢 貪欲法 | 到達可能範囲を更新するだけでOK |
| Coin Change | 貪欲法 | 🟣 DP | 貪欲だと最適解を逃す |
| Word Break | DFS | 🟣 DP | 部分問題が重なる |
